Tmux Smuginator

Feast your eyes upon my masterpiece! …Or something. Long have a looked for an up-to-date project that I could use for templating tmux sessions. This is pretty heavily based on @ThePrimeagen ’s tmux-sessionizer.sh script . I have modified it, however, to create a custom layout that I typically use for all of my tmux sessions with the awesome golang project, smug .

The Code

#!/usr/bin/env bash

if [[ $# -eq 1 ]]; then
    session_dir=$1
else
    session_dir=$(find ~/Code -mindepth 1 -maxdepth 1 -type d | fzf)
fi

session_name=$(basename "$session_dir" | tr . _)


tmux_running=$(pgrep tmux)

if [[ -z $TMUX ]] && [[ -z $tmux_running ]]; then
    smug start -f ~/.config/smug/go.yml session_name=$session_name session_dir=$session_dir
    exit 0
fi

if ! tmux has-session -t=$session_name 2> /dev/null; then
    smug start -f ~/.config/smug/go.yml session_name=$session_name session_dir=$session_dir
fi

tmux switch-client -t $session_name

But Hac, aren’t there other projects for this?

There sure are! A great example would be tmuxinator , but up until ~6 months ago, the project had gone dark. Honestly, until I started researching this blog post, I was under the impression the project was still dark. There’s also the ever popular and rock-stable Tmuxifier , but I was also under the impression that this project had gone dark. And let’s be honest, I’m a softie for anything written in Go, so, here we are. Smug, here we come!

As for how I’m calling this? Just the same as Prime is, with <ctrl> + f. This launches a fzf over my ~/Code directory, allowing me to pick from a project. From there, it does some inference on the basedirectory and uses that as the session name and launches all three tmux panes in said directory.

The future! Oooo Aaaah!

As for my future plans for this script? I’d like to get some form of project-type detection so different templates can be used, depending on the project directory specified. I’m pretty sure I can do this with cloc but I haven’t dug into it yet. For now, this will probably only end up existing in my dotfiles repo, but may in the future become it’s own project.

I’d also like to figure out a way to have config files per project, if desired, so I can customize my experience for certain projects. A great example would be my blog, where I’d like it to spin up the hugo development server possibly even launch neovim in the blog directory so I can instantly start working, removing any possible barrier for writing.

That’s all. My masterpiece is all of ~16 lines of code and I think it’s pretty great.

Until next time!
